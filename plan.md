---

## 전략 한 줄

**기능은 충분하다(묶음 A~FF). 이제 "마감 → 검증 → 출시"의 순서로, 낯선 사용자가 무너뜨릴 수 있는 모든 지점을 메운다.** 새 기능은 잔존(goal의 북극성)에 직접 기여할 때만 추가한다.

---

## 우선순위 원칙

1. **출시를 막는 것 먼저.** 코드 서명·자동 업데이트 저장소·온보딩·데이터 가져오기가 없으면 사용자가 시작조차 못 한다 → P0.
2. **떠나게 만드는 것 다음.** 크래시·흰 화면·데이터 손실·느림 → P0~P1.
3. **첫인상·신뢰는 P1.** 온보딩 품질·브랜딩·개인정보 고지.
4. **실증되지 않은 약속은 P1.** 확장 호환·다운로드 매트릭스는 "된다"가 아니라 "됨을 봤다"여야 한다.
5. **새 편의기능은 P2.** 잔존에 기여하는 것만, 가벼움·안정성을 해치지 않는 선에서.

## 단계(Phase) — 마일스톤

### Phase 0 — 검증 인프라 부팅 (지금)
문서 체계(이 5문서) + 검증 절차 확립. **앞으로 모든 라운드는 test.md의 게이트를 통과해야 status.md에 "완료"로 적힌다.**
- [x] goal/plan/status/rules/test 5문서 작성
- [ ] test.md의 스모크 테스트·회귀 체크리스트를 실제로 한 번 끝까지 돌려 현재 baseline 확정 → status.md 기록

### Phase 1 — 안정성 바닥 다지기 (P0)
출시 전, "안 죽는다"를 증명한다.
- [ ] **장시간·다탭 스트레스 세션**: 50탭 / 8시간, 메모리·크래시·흰 화면 관측 → 누수·누락 핸들러 색출
- [ ] **회귀 #1~#14 재현 테스트 스크립트화**: 각 회귀를 재현하는 최소 시나리오를 test.md에 고정, 매 빌드 확인
- [ ] **세션 복원 실증**: 강제 종료(작업관리자 kill) 후 탭·그룹·분할·스크롤·폼 복원 확인
- [ ] **에러 가시성**: uncaught/unhandledRejection·did-fail-load·preload-error 로그를 한 곳에 모아 0인지 확인

### Phase 2 — 출시 파이프라인 (P0)
낯선 사람이 설치할 수 있게 만든다.
- [ ] **제품 정체성 확정**: 이름·아이콘(ico/icns/png)·브랜드 컬러 → `electron-builder.yml`·`resources/`·디자인 토큰 반영
- [ ] **코드 서명**: Windows 서명 인증서 확보·signtool 연결. (없으면) SmartScreen 평판 축적 경로 + 사용자 안내 문서
- [ ] **자동 업데이트 실연결**: GitHub `owner/repo` 확정 → `electron-builder.yml`의 `owner` 교체 → `--publish always`로 첫 릴리즈 업로드 → 실제 업데이트 수신 확인
- [ ] **첫 릴리즈 v0.9.0 (베타)**: 설치→실행→업데이트 한 바퀴를 외부 환경(깨끗한 Windows)에서 검증

### Phase 3 — 첫인상 & 이주 (P1)
설치한 사람이 5분 안에 자기 브라우저로 만든다.
- [ ] **온보딩 화면** (`browser://welcome`): 브라우저 소개 3장 + 기본 브라우저 설정 버튼 + 검색엔진 선택
- [ ] **크롬/엣지 데이터 가져오기**: 북마크·방문기록·비밀번호·저장된 탭. (크롬 프로필 SQLite·Login Data 복호화 — Windows DPAPI)
- [ ] **기본 브라우저 등록** UX: `app.setAsDefaultProtocolClient` + OS 설정 안내
- [ ] **빈 상태 다듬기**: 새 탭·북마크·기록·다운로드의 첫 화면 안내 문구·일러스트

### Phase 4 — 약속 실증 (P1)
"된다"를 "됨을 봤다"로 바꾼다. (test.md의 매트릭스)
- [ ] **확장 호환 매트릭스 10종**: uBO Lite·Dark Reader·Bitwarden·Vimium·JSON Viewer·Stylus·Tampermonkey·Save to Pocket·Wappalyzer·ColorZilla — 로드·기본 동작 기록
- [ ] **다운로드 매트릭스**: progressive·토큰CDN octet·HLS(평문/fMP4/AES-128/master)·DASH·지원호스트·blob/MSE·쿠키게이트·이어받기 (`/audit-download`)
- [ ] **콕콕 기본 14종 실사용 점검표**: 각 기능을 실제 사이트에서 한 번씩
